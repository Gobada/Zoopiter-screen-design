# Zoopiter — Claude Code 작업 지침

이 프로젝트에서 작업할 때 항상 지킬 규칙. (한 번의 작업 지시는 Task.md 참고)

## 환경 / 빌드
- 이 프로젝트는 Figma Make에서 빌드·실행된다. 로컬 셸에 node/npm이 PATH에 없을 수 있다.
- `npm`, `npx`, `vite build`, `tsc` 등 빌드/실행 명령을 시도하지 말 것.
  실패하고 시간만 소모된다. PATH를 임의로 바꾸려 시도하지도 말 것.
- 변경 검증은 "정적으로만" 한다:
  1) 수정한 파일을 다시 읽어 문법 오류·닫히지 않은 태그가 없는지
  2) import 경로가 맞는지
  3) 요청한 변경이 정확히 반영됐는지
  를 확인하고 결과를 요약 보고한다.

## Git 반영 (중요)
- 작업이 끝나면 항상 GitHub에 push 한다. 순서:
    git add -A
    git commit -m "의미 있는 메시지"
    git push
- push 대상 브랜치는 main 으로 한다. (Claude 프로젝트 커넥터가 읽는 브랜치와 일치해야 함)
- 단계별로 진행하는 작업은 각 단계를 별도 커밋으로 만들고, 마지막에 push 한다
  (또는 각 단계 커밋 후 매번 push 해도 됨).
- git 명령은 사용 가능하다(node/npm과 달리 PATH 문제 없음). 빌드와 혼동하지 말 것.
- push 후에는 "GitHub에 반영 완료. Claude 프로젝트에서 Sync now 누르면 최신 반영됨"
  이라고 한 줄로 안내한다.

## 코드 구조
- 상수 CARD_H / OVERFLOW / DOG_W 의 단일 출처는 components/PetProfileCard.tsx.
  다른 파일은 정의하지 말고 import 해서 쓴다.
- 색·반경·그림자 등 디자인 값은 인라인에 박지 말고 theme.css 의 CSS 변수로
  정의한 뒤 참조한다. (라이트/다크 모드 둘 다 값을 둔다)
- 같은 UI를 두 곳에서 중복으로 그리지 말고 공용 컴포넌트로 재사용한다.

## 디자인 규칙
- 새 아이콘은 단색 선(line / stroke) 스타일로 만든다. 채움(fill) 아이콘 금지.
- 새 디자인은 최대한 깔끔하고 세련되게. 과한 테두리·진한 그림자 대신
  절제된 표현(부드러운 그림자, 일관된 반경)을 쓴다.

## 작업 방식
- 큰 변경은 단계별로 나누고 각 단계를 별도 커밋으로 만든다(되돌리기 쉽게).
- 기존 동작(드롭다운, 색상 선택, 사진 업로드, 애니메이션 등)은 보존한다.
- 기존 동작을 보존할 수 없거나 지시가 모호하면, 추정해서 진행하지 말고
  멈춰서 무엇이 문제인지 보고한다.
- 작업 완료 후 어떤 파일이 변경되었는지 알려준다.
